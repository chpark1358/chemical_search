# 구현 진행 기록 규칙

작성일: 2026-05-20

## 목적

이 프로젝트는 기획 문서에서 실제 구현으로 넘어가는 단계부터, 진행 단계와 구조 변경을 문서로 남긴다. 이후 새 기능을 추가하거나 구조를 바꿀 때 현재 상태를 빠르게 파악하고, 결정이 흩어지지 않게 하는 것이 목적이다.

## 필수 기록 파일

| 파일 | 역할 |
|---|---|
| `docs/project-management/progress-log.md` | 날짜별 진행 단계, 수행 작업, 검증 결과, 다음 작업 |
| `docs/project-management/structure-log.md` | 폴더/파일 구조 변경, 추가/삭제 이유, 영향 범위 |
| `docs/project-management/implementation-rules.md` | 구현 진행 기록 규칙 자체 |

## 기록 규칙

1. 새 구현 단계가 시작되면 `progress-log.md`에 단계명과 목표를 추가한다.
2. 폴더나 주요 파일이 추가/삭제/이동되면 `structure-log.md`에 남긴다.
3. 빌드, lint, 테스트, 브라우저 검증을 수행하면 결과를 `progress-log.md`에 기록한다.
4. 구현 범위가 PRD/IA와 다르게 바뀌면 사유와 영향을 `progress-log.md`에 기록한다.
5. 문서는 한국어로 작성한다.
6. 기록은 실제 변경 후 바로 업데이트한다.

## 단계 명명

- `P0-S0`: 프로젝트 scaffold와 실행 환경
- `P0-S1`: 도메인 모델과 API 기반
- `P0-S2`: 상세/라운드/플레이/결과 핵심 루프
- `P0-S3`: 탐색/카테고리 유입
- `P0-S4`: 만들기/수정
- `P0-S5`: 신고/관리자/공유/이벤트
- `P0-S6`: QA, 회귀 검증, 릴리즈 준비
