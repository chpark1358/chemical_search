# 작업 종료 보고서: Phase 1 Chemical Search CLI POC

날짜: 2026-06-10
상태: 완료
관련 단계: Phase 1 POC

## 목적

웹 UI 구현 전에 구조 정규화와 공개 compound/paper provider 검색의 핵심 흐름을 CLI로 검증한다.

## 완료한 작업

- SMILES/name/formula/InChI/InChIKey 입력 감지
- RDKit canonical SMILES, InChIKey, formula, molecular weight 정규화
- PubChem compound 후보 조회와 candidate index 선택
- ChEMBL exact/similarity/substructure 검색
- Semantic Scholar/Crossref 논문 검색
- provider 장애 시 partial result 보존
- JSON/Markdown 출력
- 모든 결과에 source URL과 match reason 표시
- 네트워크를 사용하지 않는 단위 테스트 작성

## 구현/진행 방식

외부 provider를 공통 `ProviderResult` 계약으로 감싸고 입력 감지, 정규화, 검색 orchestration, 렌더링을 분리했다. provider의 timeout, rate limit, 오류는 전체 검색 예외로 전파하지 않고 provider diagnostics와 partial 상태로 보존한다.

## 검증 결과

```powershell
$env:PYTHONUTF8='1'
py -m unittest discover -s tests -p 'test*.py' -v
py -m scripts.chemical_search.poc_cli aspirin --mode all --limit 3
py -m scripts.chemical_search.poc_cli C9H8O4 --input-type formula --mode exact --candidate-index 0
py -m scripts.chemical_search.poc_cli "CC(=O)Oc1ccccc1C(=O)O" --mode similarity --no-semantic-scholar
```

- 단위 테스트 5개 통과
- aspirin 이름 검색에서 PubChem, ChEMBL exact/similarity/substructure, Crossref 결과 확인
- Semantic Scholar HTTP 429를 partial로 보존
- formula 입력에서 후보 3개 표시 후 aspirin 선택 검색 성공
- invalid SMILES가 전체 crash 없이 partial 결과로 출력됨

## 남은 이슈

- Semantic Scholar API key 또는 fallback 정책이 필요하다.
- Crossref를 포함한 provider별 cache/throttle/retry가 필요하다.
- formula 후보 선택은 현재 CLI index 방식이며 Phase 2에서 UI/API로 바꿔야 한다.
- 결과 병합, 중복 제거, ranking은 아직 구현하지 않았다.
- 재현 가능한 Python 실행 환경을 확정해야 한다.
