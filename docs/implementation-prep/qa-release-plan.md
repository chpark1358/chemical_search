# 이상형 월드컵 MVP QA/릴리즈 계획

기준 문서:
- `docs/ideal-worldcup-prd.md`
- `docs/mvp-ia-wireframes.md`

## 1. QA 목표

- 탐색 -> 상세 -> 라운드 선택 -> 플레이 -> 결과 -> 공유 핵심 루프가 끊기지 않는지 검증한다.
- 비회원 만들기, 이미지 업로드, 비밀번호 수정, 신고/관리자 처리가 운영 가능한지 검증한다.
- 모바일 390px과 데스크톱 1440px에서 조작, 레이아웃, 성능을 확인한다.
- MVP 성공 판단을 위해 핵심 이벤트가 누락 없이 수집되는지 검증한다.

## 2. 출시 기준

| 영역 | 출시 기준 |
|---|---|
| 핵심 루프 | 비회원 사용자가 홈 진입 후 3클릭 이내 플레이를 시작하고 결과 공유까지 완료할 수 있다. |
| 플레이 안정성 | 4강/16강/32강/후보 46명 케이스가 새로고침, 중복 클릭, 느린 네트워크에서도 상태가 깨지지 않는다. |
| 모바일 조작성 | 모바일에서 16강을 터치만으로 완료할 수 있고 터치 대상은 44px 이상이다. |
| 만들기 | 제목, 후보 4명, 수정 비밀번호만으로 비회원 월드컵을 발행할 수 있다. |
| 신고/관리 | 신고 접수 후 관리자 화면에서 유지/비공개/삭제/반려 중 하나로 처리할 수 있다. |
| 공유 | 결과 공유 링크와 정사각형 이미지를 모바일/데스크톱에서 사용할 수 있다. |
| 보안 | 스크립트 입력, 위험 URL, 비허용 파일, 관리자 무권한 접근이 차단된다. |
| 성능 | 첫 화면 LCP 2.5초 이하 목표, 매치 선택 후 다음 매치 전환 체감 150ms 이하 목표 |

## 4. 데이터/권한/보안 체크

| 영역 | 체크 항목 |
|---|---|
| 세션 데이터 | `play_sessions`, `match_events`, `session_snapshots`가 같은 seed와 sessionId를 공유한다. |
| 통계 | 실제 노출 후보만 `exposure_count` 증가, bye는 승리로 집계하지 않는다. |
| 익명 사용자 | anonymous_id에 개인정보 원문을 저장하지 않는다. |
| 비밀번호 | 수정 비밀번호는 평문 저장 금지, 해시 저장, 실패 횟수 제한 적용. |
| 관리자 | `/admin` 하위 라우트는 관리자 권한 없이는 접근 불가. |
| XSS | 제목, 설명, 후보명, 신고 상세, URL 입력에 `<script>`와 이벤트 핸들러 삽입 시도. |
| 업로드 | MIME, 용량, 해상도, 확장자, EXIF 제거, 실패 후 정리 확인. |
| 외부 URL | 사설망 URL, redirect loop, 허용되지 않은 MIME 차단. |
| 신고 | 저작권/초상권/성인/혐오/개인정보/스팸/기타 사유 저장 및 처리 로그 유지. |
| 민감 콘텐츠 | 기본 탐색 노출 제외, 직접 접근 시 안내 정책 적용. |

## 6. 이벤트 로깅 검증

| 이벤트 | 필수 속성 |
|---|---|
| `worldcup_view` | worldcup_id, referrer, category |
| `search_submit` | query, filters, result_count |
| `category_click` | category_slug, source_screen |
| `card_click` | worldcup_id, card_position, source_screen |
| `round_select` | worldcup_id, selected_round |
| `play_start` | worldcup_id, selected_round, anonymous_id/user_id |
| `match_select` | session_id, round_label, winner_id, loser_id, elapsed_ms |
| `session_restore` | session_id, restore_status, match_index |
| `play_complete` | session_id, winner_id, duration_sec |
| `share_click` | worldcup_id, result_candidate_id, share_type |
| `share_image_download` | session_id, ratio, status |
| `create_start` | anonymous_id/user_id |
| `candidate_add` | draft_id, media_type |
| `validation_error` | screen, field, error_code |
| `create_publish` | worldcup_id, candidate_count, category |
| `report_submit` | target_type, target_id, reason |
| `moderation_action` | target_type, target_id, action, admin_id |

검증 기준:
- 실패한 API 호출은 성공 이벤트로 기록하지 않는다.
- 개인정보, 비밀번호, 원본 신고 상세 같은 민감 정보는 분석 이벤트에 넣지 않는다.
- 이벤트 수집 실패가 사용자 플로우를 막지 않는다.

## 7. 출시 전 수동 체크리스트

### 기능

- [ ] 홈 -> 상세 -> 라운드 선택 -> 플레이 -> 결과 -> 공유 플로우가 모바일/데스크톱에서 완료된다.
- [ ] 4강, 16강, 32강, 후보 46명 케이스가 정상 동작한다.
- [ ] 새로고침, 뒤로가기, 중복 클릭, 느린 네트워크에서 플레이 상태가 깨지지 않는다.
- [ ] 깨진 이미지, 긴 후보명, 세로/가로 이미지, GIF가 레이아웃을 깨지 않는다.
- [ ] 비회원 생성에서 필수값 누락, 후보 부족, 비밀번호 불일치가 명확히 안내된다.
- [ ] 신고 처리 후 콘텐츠의 검색/상세/플레이 노출 정책이 일관된다.

### 접근성/사용성

- [ ] 키보드만으로 상세 CTA, 라운드 선택, 플레이, 결과 공유까지 이동 가능하다.
- [ ] 후보 이미지와 주요 버튼에 접근 가능한 이름이 있다.
- [ ] 색상만으로 오류/비활성/선택 상태를 구분하지 않는다.
- [ ] 모달과 하단 시트가 focus trap과 focus 복귀를 지원한다.
- [ ] 모바일 터치 대상이 최소 44px 이상이다.

### 성능/안정성

- [ ] 첫 화면 LCP 2.5초 이하 목표를 주요 모바일 환경에서 확인한다.
- [ ] 매치 선택 후 다음 매치 전환 체감 150ms 이하 목표를 확인한다.
- [ ] 이미지 lazy loading과 thumbnail 최적화가 적용되어 목록 스크롤이 끊기지 않는다.
- [ ] API/서버/네트워크 오류에서 재시도 또는 이동 CTA가 제공된다.
- [ ] 이벤트 수집 실패가 플레이/생성/신고 플로우를 막지 않는다.

## 8. 릴리즈 차단 조건

- 플레이 완료가 불가능하거나 결과가 잘못된 우승자로 저장된다.
- `match_select`, `play_complete`, `create_publish`, `report_submit` 중 하나 이상이 정상 플로우에서 누락된다.
- 비회원 수정 비밀번호가 평문 저장되거나 관리자 경로가 무권한 사용자에게 노출된다.
- 신고로 비공개 처리한 콘텐츠가 홈/검색/상세/플레이에 계속 노출된다.
- 주요 모바일 브라우저에서 16강 완료가 불가능하다.
- 업로드/외부 URL 처리에서 스크립트 실행, 사설망 접근, 비허용 파일 저장이 가능하다.
- 공유 이미지 생성 실패가 결과 화면 전체를 막는다.

## 9. 출시 후 24시간 모니터링

- 플레이 시작률, 완료율, 평균 매치 선택 시간
- create_publish 실패율과 validation_error 상위 원인
- report_submit 수와 처리 대기 건수
- 공유 클릭률과 공유 이미지 생성 실패율
- API 5xx, 이미지 처리 실패, 세션 복구 실패
- 신고/저작권 관련 긴급 비공개 처리 필요 여부
