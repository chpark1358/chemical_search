## 1. MVP 범위

MVP는 "찾기 -> 상세 확인 -> 라운드 선택 -> 플레이 -> 결과 공유 -> 만들기 -> 신고/처리"까지의 핵심 루프를 안정적으로 구현하는 데 집중한다.

### P0 포함

- 홈/탐색/카테고리 목록
- 월드컵 상세, 후보 미리보기, 라운드 선택
- 1:1 토너먼트 플레이, 진행 상태, 새로고침 복구
- 결과, 랭킹 요약, 링크/이미지 공유
- 비회원 월드컵 생성, 후보 등록, 비밀번호 기반 수정
- 신고 접수, 관리자 신고 검토/처리
- 핵심 이벤트 로깅과 공통 오류/빈 상태

### P0 제외

- 댓글, 좋아요, 저장
- 계정 기반 이어하기/내 월드컵 관리
- 친구 결과 비교 링크
- 후보 대량 등록, CSV/TSV, URL 목록 일괄 등록
- 방송 모드, 이벤트 예측, 앱/PWA, AI 후보 생성
- 플레이 화면 광고

## 2. 마일스톤

| 단계 | 목표 | 완료 조건 |
|---|---|---|
| M1. 도메인/콘텐츠 기반 | 월드컵과 후보를 저장하고 조회할 수 있다. | 테스트 월드컵 생성, 후보 4명 이상 등록, 상세에서 후보 표시 |
| M2. 플레이 루프 | 라운드 선택 후 1:1 토너먼트를 끝낼 수 있다. | 4강/16강/32강과 후보 46명 케이스 정상 완료 |
| M3. 탐색/결과/공유 | 홈에서 찾고 결과를 공유할 수 있다. | 홈 3클릭 이내 플레이 시작, 결과 링크/이미지 생성 |
| M4. 만들기/수정 | 비회원이 월드컵을 만들고 비밀번호로 수정할 수 있다. | 제목, 후보 4명, 비밀번호만으로 발행 가능 |
| M5. 운영/출시 준비 | 신고 처리와 이벤트 로깅으로 MVP 운영이 가능하다. | 신고 콘텐츠를 유지/비공개/삭제 중 하나로 처리 가능 |

## 3. P0 사용자 스토리

| ID | 에픽 | 사용자 스토리 | 수용 기준 | 의존성 |
|---|---|---|---|---|
| P0-01 | 탐색 | 방문자로서 인기/최신 월드컵을 보고 바로 시작하고 싶다. | 홈에 검색, 만들기 CTA, 인기/최신 섹션, 카테고리 칩이 보인다. 카드에는 대표 이미지, 제목, 후보 수, 플레이 수가 표시된다. | 목록 API, 썸네일 |
| P0-02 | 탐색 | 방문자로서 검색어, 카테고리, 정렬, 기간으로 목록을 좁히고 싶다. | URL 쿼리와 필터 상태가 동기화된다. 결과 없음, 로딩, 오류 상태가 있다. | 검색 API, 카테고리 |
| P0-03 | 상세 | 방문자로서 플레이 전 후보 수와 예상 시간을 확인하고 싶다. | 상세 첫 화면에 제목, 설명, 후보 수, 플레이 수, 업데이트일, 후보 미리보기, 랭킹 Top 5, 시작 CTA가 보인다. | 상세/후보/랭킹 API |
| P0-04 | 라운드 선택 | 플레이어로서 가능한 라운드만 선택하고 싶다. | 결승/4강/8강/16강/32강 중 후보 수 이하의 2의 거듭제곱만 활성화된다. 후보 수보다 큰 라운드는 비활성화된다. | 후보 수, 라운드 계산 |
| P0-05 | 플레이 | 플레이어로서 좌우 후보를 클릭, 키보드, 모바일 터치로 선택하고 싶다. | 좌/우 카드 전체가 선택 영역이다. ArrowLeft/ArrowRight, Esc 종료 확인을 지원한다. 모바일 터치 영역은 44px 이상이다. | 플레이 UI, 세션 API |
| P0-06 | 플레이 | 플레이어로서 중복 클릭이나 네트워크 지연에도 진행 상태가 깨지지 않길 원한다. | 선택 중 입력 잠금이 적용된다. 같은 matchId/clientSequence 중복 요청은 한 번만 집계된다. | 멱등 선택 API |
| P0-07 | 플레이 | 플레이어로서 새로고침 후에도 같은 세션을 복구하고 싶다. | sessionId와 anonymous_id 기준으로 서버 snapshot을 복구한다. 실패 시 명확한 재시작 안내를 제공한다. | session_snapshots |
| P0-08 | 결과 | 플레이어로서 우승자와 선택 경로를 확인하고 싶다. | 결과 화면에 우승자 이미지/이름, 선택 브래킷 요약, 다시하기, 다른 라운드 버튼이 있다. | 완료 세션 |
| P0-09 | 랭킹 | 플레이어로서 내 우승자가 전체 랭킹에서 어느 정도인지 보고 싶다. | 전체 1위, 우승자 순위, 승률/선택률 최소 요약을 표시한다. | 후보 통계 |
| P0-10 | 공유 | 플레이어로서 결과 링크와 이미지를 공유하고 싶다. | 링크 복사, 정사각형 공유 이미지 저장을 지원한다. `share_click`이 기록된다. | 공유 이미지 생성 |
| P0-11 | 만들기 | 제작자로서 로그인 없이 제목/설명/카테고리/비밀번호를 입력하고 싶다. | 제목, 카테고리, 수정 비밀번호와 확인은 필수다. 스크립트 입력은 차단된다. | 생성 폼, 검증 |
| P0-12 | 만들기 | 제작자로서 후보 4명 이상과 이미지를 등록하고 싶다. | 후보명은 필수다. 이미지 업로드 또는 허용된 미디어 URL을 등록할 수 있다. 후보 4명 미만은 발행이 막힌다. | 업로드, 후보 API |
| P0-13 | 만들기 | 제작자로서 발행 전 상세/플레이 상태를 미리 확인하고 싶다. | 검증 요약, 대표 이미지, 가능한 라운드, 샘플 매치 미리보기가 보인다. | 발행 검증 |
| P0-14 | 수정 | 제작자로서 비밀번호로 기존 월드컵을 수정하고 싶다. | `/worldcup/:slug/edit`에서 비밀번호 확인 후 수정 화면에 진입한다. 비밀번호 분실 복구가 어렵다는 안내가 있다. | edit-auth API |
| P0-15 | 신고 | 사용자로서 상세/결과/후보를 신고하고 싶다. | 신고 사유는 저작권/초상권, 성인/선정성, 혐오/차별, 개인정보, 스팸, 기타를 지원한다. 제출 후 중복 안내가 있다. | reports API |
| P0-16 | 관리자 | 관리자로서 신고 큐를 보고 처리하고 싶다. | 신고 목록에 대상, 사유, 상태, 접수일, 처리 액션이 보인다. 유지/비공개/삭제/반려 중 하나로 처리한다. | 관리자 인증 |
| P0-17 | 이벤트 | 제품 담당자로서 핵심 퍼널 이벤트를 확인하고 싶다. | `worldcup_view`, `play_start`, `match_select`, `play_complete`, `share_click`, `create_publish`, `report_submit`이 누락 없이 기록된다. | events API |
| P0-18 | 공통 상태 | 사용자로서 오류나 빈 상태에서 다음 행동을 알고 싶다. | 404, 비공개/삭제, 서버 오류, 로딩 실패, 검색 결과 없음, 후보 부족 상태에 재시도/이동 CTA가 있다. | 공통 UI |

## 4. 릴리즈 전 필수 검증

1. 홈 -> 상세 -> 라운드 선택 -> 플레이 -> 결과 -> 공유 플로우가 모바일과 데스크톱에서 완료된다.
2. 4강, 16강, 32강, 후보 46명 케이스가 정상 동작한다.
3. 플레이 중 새로고침, 중복 클릭, 느린 네트워크에서도 결과와 통계가 깨지지 않는다.
4. 비회원 제작자는 제목, 후보 4명, 비밀번호만으로 테스트 월드컵을 발행할 수 있다.
5. 필수값 누락, 후보 부족, 비밀번호 불일치, 업로드 실패가 명확히 안내된다.
6. 신고된 콘텐츠는 관리자 화면에서 유지/비공개/삭제/반려 중 하나로 처리된다.
7. 결과 공유 이미지는 모바일 메신저와 데스크톱 다운로드에서 깨지지 않는다.

## 5. 구현 우선순위

1. DB 스키마, 미디어 업로드, 월드컵/후보 CRUD
2. 토너먼트 seed, bracket 생성, play session, match 선택 API
3. 상세, 라운드 선택, 플레이, 결과 화면 연결
4. 홈/탐색/카테고리 유입 경로
5. 만들기 3단계와 비밀번호 수정 인증
6. 공유 이미지, 신고, 관리자 처리, 이벤트 로깅

## 6. 결정 필요 사항

| 항목 | 권장안 | 결정 시점 |
|---|---|---|
| 전체 후보 모드 | 기본은 랜덤 샘플링, 전체 후보 부전승 모드는 설정 옵션 | 플레이 엔진 구현 전 |
| 공유 이미지 형식 | P0는 정사각형 1종 우선, 스토리/가로형은 후순위 | 결과 화면 구현 전 |
| 관리자 인증 | MVP 내부 운영자 계정 또는 임시 보호 라우트 | 관리자 화면 구현 전 |
| 외부 이미지 URL | 서버 검증 후 프록시/복사 저장 권장 | 업로드 구현 전 |
| 민감 콘텐츠 노출 | 기본 검색 노출 제외, 직접 접근 시 안내 | 생성/신고 구현 전 |
