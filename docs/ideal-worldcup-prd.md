# 이상형 월드컵 서비스 기획 및 PRD

작성일: 2026-05-20  
조사 범위: PIKU, Kravv, Pickiverse, PICWIN, 알빠노, 방놀, ZYUHA 및 추가 브래킷 서비스 참고

## 1. 목적

이 문서는 이상형 월드컵 사이트를 새로 개발하기 위한 제품 기획서다. 기존 서비스의 UI/UX, 콘텐츠 구조, 플레이 경험, 제작 흐름, 공유/랭킹/댓글 기능을 비교하고, 부족한 부분을 보완한 신규 서비스 요구사항을 정의한다.

목표는 단순한 1:1 선택 게임이 아니라, 사용자가 쉽게 만들고, 빠르게 플레이하고, 결과를 공유하고, 다시 돌아올 이유가 있는 취향 기반 토너먼트 플랫폼을 만드는 것이다.

## 3. 핵심 인사이트

1. 기존 한국형 이상형 월드컵의 핵심 기대치는 "목록에서 고르기 → 라운드 선택 → 1:1 선택 → 결과/랭킹/공유"로 이미 고정되어 있다.
2. 좋은 서비스는 시작 전에 전체 후보 수, 예상 소요 시간, 라운드 선택, 후보 미리보기, 참여 수를 보여준다.
3. 플레이 화면에서 가장 중요한 것은 이미지 크기, 선택 영역 명확성, 진행률, 키보드/모바일 조작, 뒤로가기/나가기 안정성이다.
4. 제작 UX는 대부분 로그인 장벽이 있어 이탈이 크다. ZYUHA처럼 비로그인 제작 + 수정 비밀번호 방식은 초기 활성화에 유리하다.
5. 랭킹은 단순 우승 횟수보다 선택률, 매치 수, 승률, 최근 상승, 내 결과와 전체 결과 비교가 있어야 재미가 커진다.
6. 광고는 수익화에는 필요하지만 CTA와 선택 화면을 방해하면 곧바로 체감 품질을 낮춘다.
7. 댓글은 재미를 만들지만 스팸, 저품질, 욕설, 정치/성인성 콘텐츠 관리가 필수다.
8. 대형 후보군(256강 이상)은 "빠른 모드", "이어하기", "중간 저장", "탈락자 보기"가 없으면 완주율이 낮아진다.

## 4. 제품 방향

### 포지셔닝

"누구나 3분 안에 만들고, 친구에게 바로 공유하며, 내 취향과 전체 랭킹을 비교하는 이상형 월드컵 플랫폼"

### 차별화 원칙

- 플레이는 PIKU처럼 빠르게, 화면 정보는 방놀처럼 명확하게, 제작은 ZYUHA처럼 낮은 장벽으로 제공한다.
- 콘텐츠 신뢰도는 Kravv의 에디터 노트 방식을 일부 도입하되, 사용자 제작 콘텐츠는 커뮤니티 지표와 신고/검수로 관리한다.
- 공유 결과는 단순 링크가 아니라 이미지 카드, 결과 페이지, 친구 비교 링크까지 제공한다.
- 비회원도 플레이/제작을 가능하게 한다. 댓글은 P1에서 비회원 닉네임/비밀번호 방식으로 열고, 저장/수정/통계 관리/팔로우는 로그인 혜택으로 둔다.

## 6. 핵심 사용자 여정

### 6.1 플레이어 여정

1. 홈에서 인기/최신/카테고리/검색으로 월드컵 발견
2. 카드에서 후보 2개 미리보기, 후보 수, 참여 수, 예상 소요 시간 확인
3. 상세 페이지에서 설명, 후보 미리보기, 종합 랭킹을 확인한다. 댓글은 P1 이후 확인한다.
4. 시작 전 라운드 선택: 결승, 4강, 8강, 16강, 32강, 전체
5. 플레이: 좌우 선택, 방향키, 모바일 스와이프/탭 지원
6. 중간 진행: 진행률, 현재 라운드, 탈락자, 남은 경기 확인
7. 결과: 우승자, 내 선택 경로, 전체 랭킹 대비, 공유 이미지 생성
8. 후속 행동: 다시하기, 다른 라운드, 유사 월드컵 추천, P1 댓글 작성

### 6.2 제작자 여정

1. 만들기 진입
2. 제목/설명/카테고리/공개 여부/수정 비밀번호 입력
3. 후보 등록: 이름, 이미지/GIF/동영상 링크, 설명, 태그
4. 대량 등록: CSV, 이미지 드래그앤드롭, URL 목록 붙여넣기
5. 토너먼트 옵션: 후보 섞기, 풀이자 라운드 선택 허용, 최소/최대 라운드
6. 미리보기와 유효성 검사
7. 발행 후 공유 링크/썸네일 자동 생성
8. 통계 대시보드에서 플레이 수, 완주율, 우승자 변화 확인

## 8. 화면별 요구사항

### 8.1 홈/탐색

필수:
- 검색창은 상단 고정 또는 첫 화면에 배치한다.
- 정렬: 인기순, 최신순, 완주율순. 급상승/댓글순은 P1에서 제공한다.
- 기간 필터: 전체, 오늘, 주간, 월간.
- 미디어 필터: 이미지, GIF, 동영상, 텍스트.
- 카테고리 탭: 아이돌/연예, 애니/게임, 음식, 스포츠, 여행, 유머, 기타.
- 카드에는 대표 이미지 2장, 제목, 한 줄 설명, 후보 수, 참여 수, 완주율, 업데이트일을 표시한다.

개선 포인트:
- PIKU의 빠른 시작성과 Pickiverse의 커뮤니티 신호를 합친다.
- 광고가 있다면 카드 그리드 사이에 자연스럽게 넣고, CTA와 플레이 선택 영역 위에는 두지 않는다.

### 8.2 상세 페이지

필수:
- 제목, 설명, 제작자, 후보 수, 플레이 수, 업데이트일.
- 좋아요와 댓글 수는 P1 이후 노출한다.
- 시작 CTA는 첫 화면에 반드시 보인다.
- 시작 라운드 선택은 CTA 근처에 둔다.
- 후보 미리보기 12~24개와 더보기.
- 실시간 종합 순위: 우승 횟수, 승률, 선택률.
- 신고/공유.
- 댓글/저장은 P1 기능으로 제공한다.

차별 기능:
- 예상 소요 시간: 16강 약 2~3분, 32강 약 5분 등.
- "내 결과와 전체 결과 비교" 미리보기.
- 저작권/부적절 신고 안내.
- 제작자 다른 콘텐츠와 유사 월드컵 추천.

### 8.3 플레이 화면

필수:
- 좌우 후보는 동일 크기와 동일 정보량으로 표시한다.
- 후보 선택 영역은 이미지 전체와 이름 영역 모두 클릭 가능해야 한다.
- 진행 정보: 라운드명, 현재 경기, 전체 진행률, 남은 경기.
- 키보드: 좌/우 선택, Esc 나가기, Space 빠른 선택 옵션.
- 모바일: 탭 선택, 좌우 스와이프, 큰 터치 영역.
- 나가기 전 확인 모달과 같은 브라우저 세션의 새로고침 복구를 지원한다.

차별 기능:
- 방놀 방식의 "탈락자", "이번 라운드 경기 큐", "전체화면" 제공.
- 긴 월드컵의 계정 기반 이어하기와 장기 임시 저장은 P1 기능으로 제공한다.
- "선택 이유 메모"는 선택 사항으로 제공해 결과 공유를 풍부하게 만든다.
- 이미지 로딩 실패 시 대체 카드와 재시도.

### 8.4 결과 화면

필수:
- 우승자 이미지/이름/설명.
- 내 선택 브래킷 요약.
- 전체 랭킹 대비: 전체 1위, 내 우승자 순위, 승률.
- 다시하기, 다른 라운드로 다시하기, 공유하기.
- P1 이후 댓글 작성 유도.

차별 기능:
- SNS 공유 이미지 자동 생성: 정사각형, 스토리형, 가로형.
- 친구 비교 링크: 같은 월드컵을 친구가 하면 결과 비교 표시.
- "나와 같은 우승자를 고른 비율" 표시.
- 결과별 추천 월드컵.

### 8.5 만들기

필수:
- 비회원 제작 허용. 단, 수정/삭제용 비밀번호 필요.
- 로그인 제작 시 자동 저장, 통계, 수정 이력 제공.
- 기본 정보: 제목, 설명, 카테고리, 태그, 공개/비공개, 성인/민감 콘텐츠 여부.
- 후보 등록: 이름, 이미지/GIF/동영상 URL, 설명.
- 최소 후보 4명, 권장 8명 이상.
- 후보 수에 따라 가능한 라운드 자동 계산.
- 미리보기와 오류 검증.

차별 기능:
- 대량 등록:
  - 이미지 여러 장 드래그앤드롭
  - 파일명으로 후보명 자동 입력
  - CSV/TSV 붙여넣기
  - URL 목록에서 이미지 가져오기
- 자동 썸네일 생성.
- 중복 후보 감지.
- 저해상도/깨진 이미지 경고.
- AI 보조는 선택 기능으로만 제공: 제목/설명 다듬기, 태그 추천, 후보 설명 생성.

## 9. 기능 요구사항

### 9.1 월드컵 플레이 엔진

- 지원 라운드: 결승, 4강, 8강, 16강, 32강, 64강, 128강, 256강, 512강, 1024강.
- 후보 수가 정확한 2의 거듭제곱이 아니면 랜덤 샘플링 또는 부전승 정책을 적용한다.
- 기본값은 랜덤 샘플링으로 하되, 제작자가 "전체 후보 최대 활용" 옵션을 켤 수 있다.
- 동일 사용자의 새로고침/뒤로가기에도 진행 상태를 복구한다.
- 선택 이벤트는 match_id, winner_id, loser_id, round, elapsed_ms로 기록한다.

### 9.1.1 대진 생성 정책

대진 생성 규칙은 출시 전에 고정해야 한다. 규칙이 모호하면 랭킹 신뢰도와 재현성이 흔들린다.

- 기본 모드: 선택한 라운드 수만큼 후보를 무작위 샘플링한다. 예를 들어 후보 46개에서 32강을 선택하면 32개만 참여한다.
- 전체 후보 모드: 가능한 경우 부전승을 허용해 모든 후보를 최대한 사용한다. 예를 들어 후보 46개는 64강 구조 안에서 일부 후보가 부전승으로 다음 라운드에 진입한다.
- 같은 세션 안에서는 대진표가 고정되어 새로고침해도 후보 순서가 바뀌지 않는다.
- 같은 사용자가 다시하기를 누르면 새 랜덤 시드로 새 세션을 만든다.
- 제작자는 기본 라운드와 최대 라운드를 지정할 수 있고, 풀이자가 시작 전 라운드를 바꿀 수 있는지 선택한다.
- 통계에는 실제 노출된 후보만 exposure_count를 증가시킨다. 부전승은 match_wins에는 포함하지 않고 finals/우승 통계에는 정상 반영한다.

### 9.2 랭킹/통계

- 후보별 지표:
  - 우승 수
  - 결승 진출 수
  - 매치 승리 수
  - 매치 패배 수
  - 승률
  - 선택률
  - 최근 24시간 순위 변화
- 월드컵별 지표:
  - 조회 수
  - 시작 수
  - 완주 수
  - 완주율
  - 평균 소요 시간
  - 공유 수
  - 댓글 수

### 9.3 검색/발견

- 검색 대상: 제목, 설명, 후보명, 태그, 제작자명.
- 검색 결과는 카테고리/미디어/후보 수/기간/정렬로 필터링한다.
- P1 급상승은 최근 24시간 시작 수 증가율과 완주율을 함께 반영한다.
- 민감/성인 콘텐츠는 기본 검색에서 제외하고 설정에서 표시 허용한다.

### 9.4 커뮤니티

P1 기능으로 구현한다.

- 비회원 댓글 허용, 닉네임/비밀번호 방식.
- 로그인 댓글은 프로필과 연결.
- 댓글 정렬: 최신순, 인기순, 결과 공유순.
- 스팸/욕설/개인정보 자동 필터.
- 신고: 저작권, 성인/선정성, 혐오/차별, 개인정보, 스팸, 기타.

### 9.5 공유

- 링크 복사.
- 카카오톡/ X / 페이스북 / 커뮤니티용 이미지 저장.
- 결과 카드 자동 생성.
- 임베드 위젯은 2차 개발 범위로 둔다.

### 9.6 방송 모드

MVP 이후 확장:
- 큰 글씨와 전체화면.
- 선택 제한 시간.
- 시청자 투표 집계.
- OBS 브라우저 소스용 경량 화면.
- 채팅 명령어 연동은 플랫폼 정책 검토 후 진행.

## 12. MVP 범위

기존 경쟁 서비스와 차별화하려면 기능을 넓게 넣는 것보다 첫 출시에서 "잘 찾고, 바로 플레이하고, 쉽게 만들고, 예쁘게 공유한다"를 안정적으로 완성하는 편이 낫다. 따라서 MVP는 P0/P1로 나누어 출시 리스크를 낮춘다.

### P0: 첫 출시 필수
- 홈/탐색
- 월드컵 상세
- 플레이 엔진
- 결과/랭킹
- 비회원 제작
- 후보 이미지 업로드
- 기본 검색/정렬/카테고리
- 링크 복사와 결과 공유 이미지
- 신고 접수
- 관리자 기본 신고 처리
- 플레이/완주/공유/제작 이벤트 로깅

### P1: MVP 직후 1차 개선
- 댓글
- 좋아요/저장
- 제작자 통계 대시보드 기본형
- 후보 대량 등록
- 결과 비교 링크
- 급상승 정렬
- 검색 품질 개선

### MVP에서 제외 가능
- 앱 출시
- 방송 채팅 투표
- 소셜 로그인 전체
- 친구 비교 고급 통계
- 임베드 위젯
- AI 후보 생성
- 유료 제작자 기능
- 고급 추천 알고리즘

## 12.1 출시 전 수용 기준

P0 출시 전에 다음 기준을 만족해야 한다.

- 비회원 사용자가 홈 진입 후 3클릭 이내로 플레이를 시작할 수 있다.
- 16강 플레이는 모바일에서 한 손 조작으로 완료할 수 있다.
- 플레이 중 새로고침해도 같은 세션을 복구하거나 명확한 재시작 안내를 제공한다.
- 후보 이미지가 깨져도 선택 진행이 막히지 않는다.
- 비회원 제작자는 제목, 후보 4개, 비밀번호만으로 테스트 월드컵을 발행할 수 있다.
- 신고된 콘텐츠는 관리자 화면에서 비공개, 반려, 유지 중 하나로 처리할 수 있다.
- 결과 공유 이미지는 모바일 메신저에서 잘리지 않는 비율로 생성된다.
- 광고가 플레이 화면, 라운드 선택 모달, 결과 CTA를 덮지 않는다.

## 14. 운영 정책

- 저작권 이미지/연예인 이미지 사용은 사용자 업로드 책임 고지와 신고 기반 삭제를 기본으로 한다.
- 미성년자/성인성/혐오/정치 선동 콘텐츠는 별도 정책을 둔다.
- 후보명에 실명/개인정보가 포함될 경우 신고 시 빠르게 비공개 처리한다.
- 랭킹 조작 방지를 위해 같은 anonymous_id/IP/세션의 반복 플레이는 통계 반영 가중치를 낮춘다.
- 공개 콘텐츠는 검색 노출 전 기본 자동 검사를 거친다.

### 14.1 권리/저작권 처리

- 업로드 시 "내가 사용할 권리가 있거나, 신고 시 삭제될 수 있음을 이해한다" 동의를 받는다.
- 원저작자/초상권자 신고 채널을 명확히 제공한다.
- 반복 저작권 신고가 누적된 제작자는 공개 발행을 제한한다.
- 외부 이미지 URL을 직접 hotlink하지 않고 서버에서 썸네일을 생성하거나 프록시 정책을 둔다.
- 삭제된 후보가 포함된 기존 플레이 결과는 후보명을 비공개 처리하고 통계만 유지한다.

### 14.2 민감 콘텐츠 기준

- 성인/선정성 콘텐츠는 기본 검색과 홈 추천에서 제외한다.
- 실존 일반인, 미성년자, 학교/직장 등 식별 가능한 개인을 대상으로 한 외모 평가형 월드컵은 신고 시 우선 비공개 처리한다.
- 정치인/공인 콘텐츠는 허용하되 혐오, 허위사실, 폭력 선동 표현은 제한한다.
- 후보 설명과 댓글 모두 같은 필터 정책을 적용한다.

### 14.3 수익화 원칙

수익화는 플레이 몰입을 훼손하지 않는 범위에서 단계적으로 적용한다.

- MVP: 상세 페이지 하단/목록 중간 광고만 허용하고 플레이 화면 광고는 금지한다.
- Phase 2: 제작자 후원 링크, 프리미엄 공유 카드, 광고 제거 옵션을 검토한다.
- Phase 3: 브랜드 월드컵, 이벤트 캠페인, 임베드 위젯 유료화, 제작자 분석 기능을 검토한다.
- 광고 노출은 완주율, 이탈률, CTA 클릭률을 기준으로 A/B 테스트한다.

### 14.4 SEO/유입 전략

- 모든 공개 월드컵은 검색 가능한 상세 URL과 메타 타이틀/설명을 가진다.
- 후보명 일부를 구조화 데이터 또는 페이지 본문에 노출해 롱테일 검색을 확보한다.
- "음식 이상형 월드컵", "아이돌 이상형 월드컵 만들기" 같은 카테고리 랜딩 페이지를 만든다.
- 결과 페이지는 기본적으로 noindex 처리하고, 상세/랭킹/카테고리 페이지를 SEO 중심으로 운영한다.
- 오래된 콘텐츠는 최근 플레이/수정일 기준으로 재노출하되, 최신성과 저작권 위험을 함께 본다.

## 15. 권장 UX 디테일

- 카드 CTA는 "시작"과 "랭킹"을 분리하되, 카드 전체 클릭은 상세로 보낸다.
- 시작 버튼은 상세 첫 화면과 하단 sticky 영역에 모두 둔다.
- 라운드 선택은 모달보다 상세 사이드 패널/하단 시트로 제공해 모바일에서 자연스럽게 보이게 한다.
- 플레이 중에는 광고를 노출하지 않는다. 광고는 홈/상세/결과에 제한한다.
- 후보명은 이미지 하단 오버레이로 표시하되, 긴 이름은 2줄까지 허용하고 말줄임 처리한다.
- 결과 공유 이미지는 사용자가 별도 편집 없이 바로 올릴 수 있게 만든다.
- P1 댓글 출시 시 "내 결과 공유" 템플릿을 제공한다.

- 앱 또는 PWA

## 17. 우선순위 제안

가장 먼저 만들어야 할 것은 플레이 경험이다. 이상형 월드컵은 콘텐츠 양보다 선택 화면의 반응성과 결과 공유가 재방문을 만든다.

1. 플레이 엔진과 모바일 선택 UX
2. 비회원 제작과 후보 등록 UX
3. 상세 페이지와 결과/랭킹
4. 검색/카테고리/홈 탐색
5. 신고/관리자, 이후 댓글
6. 고급 통계/방송/친구 비교

## 18. 핵심 결론

신규 서비스는 PIKU의 콘텐츠 폭, Kravv의 설명력, Pickiverse의 커뮤니티 신호, 방놀의 플레이 정보 설계, ZYUHA의 제작 편의성을 조합해야 한다. 특히 경쟁 우위는 "비회원도 쉽게 만들 수 있음", "플레이 중 정보가 명확함", "결과 공유가 예쁨", "랭킹 비교가 재미있음"에서 나온다.

따라서 MVP는 거대한 기능보다 빠른 플레이, 쉬운 제작, 강한 공유 루프에 집중하는 것이 맞다.

## 19. 남은 의사결정

구현 전에 아래 항목은 결정이 필요하다.

| 항목 | 권장안 | 이유 |
|---|---|---|
| 비회원 제작 허용 범위 | 공개 발행 허용, 수정/삭제 비밀번호 필수 | 초기 콘텐츠 확보에 유리함 |
| 댓글 출시 시점 | P1 | MVP 운영 리스크를 줄이고 플레이/제작 품질에 집중 |
| 기본 대진 정책 | 랜덤 샘플링 | 사용자가 이해하기 쉽고 통계 계산이 단순함 |
| 전체 후보 모드 | 제작자 선택 옵션 | 대형 후보군 제작자의 요구를 수용 |
| 광고 적용 시점 | 트래픽 확인 후 | 초기 완주율과 공유율을 먼저 확보 |
| 로그인 방식 | 이메일 우선, 소셜 로그인은 후속 | MVP 구현 범위 축소 |
| 성인/민감 콘텐츠 | 기본 비노출 | 앱/광고/검색 유입 리스크 방지 |

## 20. 주요 리스크와 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| 저작권/초상권 신고 증가 | 콘텐츠 비공개, 서비스 신뢰도 하락 | 신고 플로우, 반복 위반 제한, 썸네일 프록시 |
| 저품질 월드컵 대량 생성 | 홈 품질 저하 | 기본 검색 노출 전 자동 검사, 급상승/완주율 기반 노출 |
| 대형 월드컵 완주율 저하 | 결과/공유 감소 | 기본 라운드 추천, 이어하기, 예상 소요 시간 표시 |
| 랭킹 조작 | 커뮤니티 신뢰도 하락 | 세션/IP/anonymous_id 기반 가중치와 이상치 탐지 |
| 이미지 로딩 비용 증가 | 성능/비용 악화 | 업로드 제한, 리사이징, CDN, lazy loading |
| 광고 과다 | 이탈률 증가 | 플레이 화면 광고 금지, 광고 A/B 테스트 |
| 비회원 비밀번호 분실 | 수정/삭제 CS 증가 | 로그인 전환 유도, 비회원 콘텐츠 관리 안내 강화 |

## 21. MVP QA 체크리스트

출시 직전에는 아래 플로우를 실제 기기 기준으로 확인한다.

- 데스크톱/모바일에서 홈 → 상세 → 라운드 선택 → 플레이 → 결과 → 공유까지 끊기지 않는다.
- 4강, 16강, 32강, 후보 수가 애매한 46개 케이스가 모두 정상 동작한다.
- 새로고침, 뒤로가기, 중복 클릭, 느린 네트워크에서도 대진 상태가 깨지지 않는다.
- 깨진 이미지, 긴 후보명, 세로 이미지, 가로 이미지, GIF/동영상 후보가 레이아웃을 깨지 않는다.
- 비회원 제작에서 필수값 누락, 후보 4명 미만, 비밀번호 불일치, 중복 후보가 명확히 안내된다.
- 신고 처리 후 해당 콘텐츠가 홈/검색/상세/플레이에서 의도한 상태로 보인다.
- 결과 공유 이미지는 카카오톡, 모바일 브라우저 저장, 데스크톱 다운로드에서 잘리지 않는다.
- 관리자 없이도 자동 필터가 최소한의 성인/스팸/스크립트 입력을 차단한다.
